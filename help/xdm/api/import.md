# Import endpoint

The `/rpc/import` endpoint in the [!DNL Schema Registry] API allows you to create Experience Data Model (XDM) resources from generated export payloads. Export payloads can be created from two sources:

* The [`/rpc/export` endpoint](./export.md) creates export payloads from existing XDM resources, allowing you to share resources between sandboxes.
* The [`/rpc/csv2schema` endpoint](./csv-to-schema.md) creates export payloads from CSV templates.

Once you create an export payload, you can use the `/rpc/import` endpoint to generate the resource (and all dependent resource) in the sandbox of your choice.

## Getting started

The `/rpc/import` endpoint is part of the [[!DNL Schema Registry] API](https://www.adobe.io/experience-platform-apis/references/schema-registry/). Before continuing, please review the [getting started guide](./getting-started.md) for links to related documentation, a guide to reading the sample API calls in this document, and important information regarding required headers that are needed to successfully make calls to any Experience Platform API.

The `/rpc/import` endpoint is part of the remote procedure calls (RPCs) that are supported by the [!DNL Schema Registry]. Unlike other endpoints in the [!DNL Schema Registry] API, RPC endpoints do not require additional headers like `Accept` or `Content-Type`, and do not use a `CONTAINER_ID`. Instead, they must use the `/rpc` namespace, as demonstrated in the API calls below.

## Import a resource {#import}

Once you have generated an export payload for an XDM resource, you can use that payload in a POST request to the `/import` endpoint to import that resource into a target organization and sandbox.

**API format**

```http
POST /rpc/import
```

**Request**

The following request takes the payload returned from a call to the [`/rpc/export` endpoint](./export.md) to import a field group (`Restaurant`) into a new organization and sandbox, as determined by the `x-gw-ims-org-id` and `x-sandbox-name` headers, respectively.
